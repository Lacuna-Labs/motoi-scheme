## Chapter summary

1. **Numbers, tensors** — floats, IEEE 754, tensor as shape + numbers, lists as 1-D tensors, sums.
2. **Vectors, dot product** — `vec/add`, `vec/dot`, `vec/norm`, cosine similarity, nearest-neighbor search in small vocab.
3. **Matrices, matvec** — matvec as per-row dot product, identity + scale + shear, matmul, two-layer sketch.
4. **Functions of functions** — closures, function composition, function-list chain, parameterized net builder.
5. **What is a model** — line, loss, MSE, quadratic, universal approximator hint, grid search.
6. **Gradient descent** — finite-difference slope, single-parameter descent, learning rate blowup, two-param, line-fit.
7. **Single neuron** — activations (ReLU/sigmoid/tanh), linear classifier, gradient-descent fit, XOR limitation.
8. **A layer, then two** — parallel neurons, XOR with hidden layer, three-layer network, parameter counting.
9. **Backprop** — chain rule derivation, mechanical backward pass, one-neuron backprop, two-layer MLP backprop, gradient check.
10. **Training loop** — SGD, batching, MLP on `x^2`, overfitting, early stopping, template.
11. **Tokens and embeddings** — whitespace tokenizer, vocab, one-hot, embedding table, similarity search.
12. **Attention** — softmax, dot-product attention by hand, wrap as function, multi-query, QKV projections.
13. **Transformer block** — layernorm, feed-forward, residuals, full block, pre-norm vs post-norm, parameter count.
14. **Positional encoding + masking** — sinusoidal PE, add positions, causal mask, verify masking, bidirectional note.
15. **Stacking + tokenization** — stacked blocks, why deeper helps, BPE algorithm + trainer, tokenize-BPE.
16. **The tiny LLM** — full assembly, hyperparameters, forward pass, cross-entropy loss, sampling, honest scope disclosure.

Appendix A — Math prerequisites (numbers, functions, derivatives, partial deriv, vectors, matrices, exp/log, softmax, cross-entropy, chain rule).
Appendix B — Philosophy of the field (Turing → McCulloch/Pitts → Rosenblatt → Minsky → RHW → LSTM → LeNet → AlexNet → word2vec → Bahdanau → transformer → GPT).
Appendix C — Why LLMs hallucinate (mechanism as probability distribution over vocab, sources, how to catch, meta-honesty).
Appendix D — What you built vs. what I am (mechanics same, scale different, honest inventory of Motoi Copilot capabilities).

**Caveats (documented in book cover):**

- `(vec/make (list ...))` pattern is NOT how Motoi's `vec/make` works. `vec/make` is variadic. Book uses `vec/make` as a conceptual marker; the runtime accepts lists directly for vec ops.
- Multi-list `map` (e.g., `(map + xs ys)`) is NOT supported in Motoi. Book has been fixed in layernorm to use `iota` + `list-ref`. A few chapter-10 backward-pass snippets retain multi-list `map` syntax as SHAPE, not runnable.
- `math/sin` / `math/cos` in book — Motoi ships these under `geom/sin` / `geom/cos`. Substitution documented in cover.
- `1e-5` scientific-notation numbers — Motoi doesn't parse. Book fixed to decimal (`0.00001`).
- Chapter 16 full analytic backprop is SKETCHED not fully implemented — 300-line manual backprop through the transformer would exceed the pedagogical form. Chapter is honest about this scope limit and points at chapter 9 for the mechanics.
