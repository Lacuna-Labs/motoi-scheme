# SLAT-TO-MD-CONVERTER-DESIGN-2026-07-17

> **Audience:** :alfred, :engineering, :docs-pipeline
> **Dialect:** `:motoi-eng`
> **Training-eligible:** false
> **Confidentiality:** `:internal-eng`

## Section 0 — Purpose

PURPOSE.

SLAT is the source of truth. MD is rendered build output. Every canonical
document (book, engineering doc, doctrine memory, graph record) is
authored as .slat or .slatl. The MD file that sits alongside it is
regenerated by this converter and is disposable. If MD is edited by
hand it drifts; the SLAT is what regenerates.

Alfred, 2026-07-17 evening (verbatim): 'I don't wanna be making ninety
nine thousand MD files. That was an oversight because of my trying to
be progressive thinking. The SLAT is the source of truth the MD files
get rendered from it, but I wouldn't think of it that way. It will
always just kinda just be a sidecar.'

This converter reifies the sidecar. It is one-way, idempotent, and
convention-based. No round-trip. No hand-authoring MD.

## Section 1 — Cli

CLI.

Command:  slat-render <path>
Alias:    scripts/slat-render.mjs <path>

Modes:
  slat-render foo.slat
      → renders foo.md alongside foo.slat
  slat-render foo.slatl
      → renders foo.md alongside foo.slatl
  slat-render --tree ROOT
      → walks ROOT recursively, renders every .slat/.slatl found,
        skipping _archive/, _recycling-bin*/, node_modules/, .git/
  slat-render --out DIR foo.slat
      → renders to DIR/foo.md (mirroring path structure if --tree)
  slat-render --watch [--tree ROOT | file.slat]
      → re-render on save. Uses fs.watch (no external deps).
  slat-render --quiet
      → suppress per-file logging; only print errors
  slat-render --check <path>
      → parse-only mode; exit 1 if any file fails to parse. No writes.

Exit codes:
  0 — success (all files parsed and rendered)
  1 — parse error on at least one file (with :strict or --check)
  2 — bad usage / unknown flag

Note: default behavior is TOLERANT — unknown record types render as
fenced lisp blocks with an HTML comment warning. Parse errors on
individual records also render as fenced lisp with warning; the file
still renders. --strict escalates parse errors to exit 1.

## Section 2 — Record Types

RECORD TYPES AND RENDERING CONVENTIONS.

The renderer walks the parsed record list and dispatches on the head
symbol (_form). Convention table:

── (chapter :book B :chapter-number N :title T :prose P ...) ──
   Renders as:
     (YAML frontmatter with :book :chapter-number :title :audience
      :dialect :training-eligible :confidentiality)
     # Chapter N — T
     <prose, verbatim>
   Chapter-number may be numeric or "01" style — preserved as-is.
   If the prose contains its own '# ...' heading matching the title
   the renderer detects and does not double-emit.

── (section :book B :number N :title T :short S :prose P ...) ──
   Renders as:
     (YAML frontmatter with book/number/title/short/audience/dialect)
     # §N — T
     > S       ← the :short field as a callout
     <prose, verbatim>

── (rules :collection C :version V :purpose P ...) ──
   Renders as:
     (YAML frontmatter with collection/version/owner/created)
     # Rules — <collection>
     Then each keyword field becomes a '## <keyword>' section, with
     the value rendered as:
       string → paragraph (verbatim)
       list of records → bulleted list
       list of primitives → bulleted list

── (dialect :applies (...) :note "...") ──
   Renders as a callout badge at top of doc:
     > **Dialect:** applies to `motoi`, `sakura` — <note>

── (slat-set :doc-id ID :status S :section-N-name PROSE ...) ──
   The engineering-doc top-level shape. Renders as:
     (YAML frontmatter with all short metadata fields)
     # <doc-id>
     Then each :section-N-<name> field becomes a '## Section N — <name>'
     with the triple-quoted prose rendered verbatim.

── (prose "text") or (paragraph "text") ──
   Verbatim paragraph.

── (code :dialect D :source S) or (example ...) ──
   Renders as:
     ```<dialect or motoi>
     <source>
     ```

── (cross-ref :to URL :text T) ──
   Renders as inline markdown link: [T](URL)

── (facets :name F :type T :values (...)) ──
   Renders as a markdown table with columns Name / Type / Values.

── (provenance :source S :date D :notes N) ──
   Collected across the doc and emitted as a Provenance footnote
   section at the bottom.

── (metadata :key V ...) ──
   Merged into YAML frontmatter at the top.

── (audience ...) / (training-eligible ...) / (dialect-tag ...) ──
   Rendered as callout badges under the H1:
     > **Audience:** operator | engineering
     > **Training-eligible:** yes
     > **Dialect:** motoi

── Unknown record type ──
   Fenced lisp block with an HTML comment:
     <!-- warning: unknown record type "X" — rendered raw -->
     ```lisp
     (X ...)
     ```

## Section 3 — Directory Convention

DIRECTORY CONVENTION.

Default: for a source at X/foo.slat, MD lands at X/foo.md.
For a source at X/bar/baz.slatl, MD lands at X/bar/baz.md.

With --out DIR: source X/bar/baz.slat, output DIR/bar/baz.md
(mirroring the tree structure relative to the tree root when
--tree is set). Without --tree, --out DIR flattens: DIR/baz.md.

With --tree ROOT: walks ROOT recursively. Skips:
  - any dir named _archive/, _recycling-bin/, _recycling-bin*/
  - node_modules/
  - .git/
  - dist/
  - build/
  - any dir whose name starts with .
These skips prevent re-rendering historical/archived material.

Filename conventions:
  foo.slat   → foo.md
  foo.slatl  → foo.md
  foo.ENG.slat → foo.ENG.md
  foo.book.slatl → foo.book.md
The extension is stripped and .md appended. Any dot-suffixes before
.slat/.slatl are preserved.

## Section 4 — Idempotency

IDEMPOTENCY.

Re-running the renderer against unchanged SLAT sources produces
byte-identical MD. Guarantees:

  - No timestamps injected into rendered output. If the SLAT source
    has :ts or :date fields, those are copied through verbatim. The
    renderer NEVER stamps 'Generated at <now>' into MD.
  - YAML frontmatter keys emitted in a fixed canonical order (see
    §5). Non-canonical keys emitted alphabetically after canonical.
  - Prose passed through verbatim including whitespace.
  - Trailing newline always present, single '\\n'.

This lets us diff rendered MD in CI to catch prose-drift. And it
lets docs sites (VitePress, Astro) hash the rendered output cleanly.

## Section 5 — Yaml Frontmatter

YAML FRONTMATTER.

For each rendered MD file, emit YAML frontmatter at the top with
metadata fields extracted from the SLAT. Canonical key order:

  ---
  doc-id: <if present>
  title: <if present>
  chapter-number: <if chapter>
  section-number: <if section>
  book: <if book chapter>
  author: <if present>
  date: <if :ts or :date present>
  status: <if present>
  audience: <flattened list>
  dialect: <if present>
  training-eligible: <bool>
  confidentiality: <if present>
  provenance: <if present, truncated to 200 chars with '...'>
  source-slat: <relative path from source root>
  generated: 'do not hand-edit — rendered from SLAT'
  ---

The `generated:` line is the one exception to the no-timestamp rule.
It contains no date — just the fixed string. It exists so a human
opening the MD by accident understands not to edit it.

## Section 6 — Round Trip

ROUND-TRIP.

NOT required. This converter is one-way: SLAT → MD. If someone edits
the MD and it drifts from SLAT, the next render overwrites the MD.
We accept this because SLAT is the source of truth.

However: for docs-site integrations that need to write back (e.g.
a VitePress commit hook that captures a metadata edit), we reserve
YAML frontmatter as the auxiliary channel. A future 'md-to-slat-
frontmatter-patch' tool could apply frontmatter edits back to the
SLAT. Out of scope for this ticket.

## Section 7 — Errors

ERROR HANDLING.

Level 1 — unknown record type. Emit a fenced lisp block with an
HTML warning comment. Do not crash. Do not exit non-zero unless
--strict.

Level 2 — parse error on a single record. Emit the raw SLAT text
for that record in a fenced lisp block with warning. Do not crash.

Level 3 — parse error on the whole file (e.g. unbalanced parens).
Print a diagnostic to stderr with file:line:col and skip the file.
Exit 1 at end if any file failed. --strict escalates any parse
error to exit 1 immediately.

Level 4 — I/O error (unreadable source, unwritable target). Print
diagnostic, exit 2.

All warnings are also visible in the rendered MD as HTML comments,
so a reader (or a CI diff) can catch degraded rendering.

## Section 8 — Watch Mode

WATCH MODE.

--watch uses Node's fs.watch() (no external deps like chokidar).
Recursive: true on macOS + Linux. On file change events, debounce
200ms then re-render the affected file (or full tree if --tree).

Print one line per re-render:
  [slat-render] 2026-07-17T23:12:04Z rendered X/foo.slat → X/foo.md

Ctrl-C exits cleanly.

## Section 9 — Parser Notes

PARSER NOTES.

SLAT files use two conventions in the wild:
  (a) Line-oriented .slat — one form per line (used by mailboxes,
      graph edges, index files). The JS slat.js reader handles this.
  (b) Pretty-printed multi-line .slat / .slatl — one large form
      spanning many lines with indented keyword fields (used by
      engineering docs, book chapters, doctrine files). The JS
      slat.js reader does NOT handle this — it throws on newline.

This prototype ships a tolerant multi-line S-expression reader
bundled inline. It handles:
  - line and block comments (; ... and #| ... |#)
  - strings with escapes ("..." and triple-quoted \"\"\"...\"\"\")
  - #inst "..." reader macros (passed through as tagged value)
  - #t / #f / nil
  - :keywords
  - [:kw ...] vector literals
  - #0= / #0# structural sharing (best-effort; passed through)
  - integers, floats, rationals, symbols
  - nested forms across newlines

Triple-quoted strings are a doc-authoring convention seen in .ENG.slat
files — they were adopted because prose with newlines and quotes is
awful to escape. The parser recognizes \"\"\"...\"\"\" and preserves
content verbatim. Not R7RS-canonical but widespread in the tree.

When the parser encounters a form it cannot parse, it falls back to
slurping until balanced-parens and emits a _bad-form record with the
raw text — same fallback as slat_reader.py.

## Section 10 — Ci Integration

CI INTEGRATION (recommendation).

Recommended CI job (motoi-scheme, sakura-scheme, curator, lacuna-labs):

  npm run slat-render:check   # runs slat-render --tree . --check
    → fails CI if any .slat / .slatl file cannot be parsed.

  npm run slat-render:build   # runs slat-render --tree . --out dist/docs
    → produces rendered MD for VitePress / docs site consumption.

Pre-commit hook (optional): auto-render on .slat / .slatl commit,
auto-stage the .md sidecar. See scripts/install-hooks.sh for the
pattern used by build:cart-index in curator.

Non-goal: do NOT commit rendered MD by default in motoi-scheme.
Add .md to .gitignore for authored .slat / .slatl paths. Docs sites
render at deploy time. Exception: doctrine memory files in
~/.claude/projects/ remain hand-authored MD — those are not SLAT-
sourced (see §11).

## Section 11 — What Is Not Rendered

WHAT IS NOT RENDERED.

Explicit skip list — these MD files stay hand-authored:
  - ~/.claude/projects/**/*.md — Claude Code doctrine memory system
    reads MD frontmatter. Not SLAT-sourced. Do not touch.
  - README.md files at repo roots — human-facing quickstart, MD is
    the right substrate.
  - CLAUDE.md / AGENTS.md — agent-instruction files, MD substrate.
  - CHANGELOG.md — hand-authored.
  - Third-party bundled docs under vendor/ or node_modules/.

Everything else authored as SLAT gets rendered.

## Section 12 — Prototype Scope

PROTOTYPE SCOPE (v0.1 shipped 2026-07-17).

Covers:
  - Multi-line S-expression parser (bundled inline, no deps)
  - Triple-quoted string handling
  - Record types: chapter, section, rules, slat-set, dialect,
    convention, prose, code, cross-ref, metadata, provenance,
    example, facets
  - CLI: single file, --tree ROOT, --out DIR, --check, --strict,
    --quiet
  - Unknown record fallback (fenced lisp with warning)
  - YAML frontmatter emission
  - Idempotent output

Deferred to v0.2:
  - --watch mode (design in §8; implementation deferred)
  - Cross-graph link resolution (e.g. [[doctrine-name]] → URL)
  - Facet-table rendering with per-facet enumerated values
  - Structural-sharing round-trip (#0=/#0# passed through as opaque)
  - Rosetta translation between dialect tag vocabularies
